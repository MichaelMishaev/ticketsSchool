## 🎯 The 3-Second Decision

```
Need to FIND something? ────➤ 🔵 BLUE ($0.20)
Need to WRITE code? ────────➤ 🟢 GREEN ($1)
Need DEEP analysis? ────────➤ 🔴 RED ($5) - RARE!
```

---

## 🔵 Blue Agents (Use Often!)

| Agent | Use For | Example |
|-------|---------|---------|
| **blue-schema-explorer** | DB models & relations | "Find Event model fields" |
| **blue-security-scanner** | Security vulnerabilities | "Check for data leaks" |
| **blue-registration-finder** | Registration logic | "Find capacity enforcement" |

**Cost:** $0.20 each | **Speed:** 5-15 seconds | **Use:** 60% of tasks

---

## 🟢 Green Agents (Use Daily!)

| Agent | Use For | Example |
|-------|---------|---------|

| **green-form-builder** | React forms (RTL) | "Build registration form" |
| **green-bug-fixer** | Fix bugs + document | "Fix session cookie bug" |
| **green-test-writer** | Playwright tests | "Test registration flow" |

**Cost:** $1 each | **Speed:** 20-60 seconds | **Use:** 35% of tasks

---

## 🔴 Red Agent (Use Rarely!)

| Agent | Use For | Example |
|-------|---------|---------|
| **red-architect** | Critical decisions | "Security audit for launch" |

**Cost:** $5 each (25x Blue!) | **Speed:** 2-5 minutes | **Use:** 5% of tasks

⚠️ **Before using Red, ask:** Can Blue or Green handle this?

## 🏆 Golden Rules

1. **Blue first** - Search before you code
2. **Green for code** - Let agents write production code
3. **Red rarely** - Only for critical decisions
4. **Parallel Blues** - Run 🔵🔵🔵 together (fast!)
5. **Document everything** - Agents auto-document

## 🔥 Pro Tips

### Tip #1: Parallel Blues
```bash
# Instead of sequential:
"Use blue-schema-explorer" → wait
"Use blue-security-scanner" → wait

# Do parallel:
"Use blue-schema-explorer, blue-security-scanner, and blue-registration-finder in parallel"

```
